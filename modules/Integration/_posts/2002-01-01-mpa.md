---
title: "Mergin MetaPhlAn"
---


MetaPhlAn ships with a utility called `merge_metaphlan_tables.py`.

This utility takes several per-sample MetaPhlAn profiles and joins them into a single abundance table 
(taxa as rows, samples as columns), filling missing taxa with `0`.

## What it expects as input

Standard MetaPhlAn output files, i.e. the ones with `#`-prefixed header lines. It reads:

- The **first** header line's first field → the MetaPhlAn version string.
- The **last** header line (split on `#`) → the column names.
- Then columns `0` and `2` (clade name + `relative_abundance`) for normal profiles, or columns `0` and `1` for GTDB profiles.

So it merges **relative abundances only** — read counts / extra columns from `rel_ab_w_read_stats` are dropped.

## Basic usage

Pass profiles as positional arguments and redirect stdout:

```bash
./merge_metaphlan_tables.py sample1_profile.txt sample2_profile.txt sample3_profile.txt > merged_abundance.tsv
```

Or with a wildcard (handy after a taxprofiler / MetaPhlAn4 batch run):

```bash
./merge_metaphlan_tables.py profiles/*_profile.txt > merged_abundance.tsv
```

Using `-o` instead of a shell redirect (adds an overwrite guard):

```bash
./merge_metaphlan_tables.py profiles/*.txt -o merged_abundance.tsv
```

## Options

| Flag | Purpose |
|------|---------|
| *(positional)* | One or more MetaPhlAn tables to join |
| `-l FILE` | Read the list of input paths from a file (one path per line; only the first whitespace-delimited token per line is used) |
| `-o FILE` | Write to `FILE` instead of stdout |
| `--overwrite` | Allow `-o` to clobber an existing file |
| `--gtdb_profiles` | Use when inputs are GTDB-based profiles (`;` separator, different columns) |
 

## Output

A TSV with the version string on line 1, then a header row of sample names, then one row per taxon. Rows are sorted by taxonomic depth (fewest `|` levels first, so kingdom-level rows appear before species/SGB rows).

## Gotchas worth flagging to workshop participants

- **Same version only.** It builds a set of version strings from the first header line and aborts if it sees more than one. Mixing MetaPhlAn 4.0 and 4.1 profiles in the same merge will fail — reprofile with one version.
- **Sample names come from filenames.** It takes the basename, strips the extension, and removes a `_profile` suffix. So `SRR000001_profile.txt` → column `SRR000001`. Watch for name collisions if profiles from different runs share stems.
- **Headers are mandatory.** If a file has no `#` lines it errors out with "no headers … or improperly formatted." A common cause is an empty/failed profile, or a file that was post-processed and lost its header.
