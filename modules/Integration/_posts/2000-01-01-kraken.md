It all start with a set of Kraken reports, each in this format:

```text
100.00    100000       0    R    1      root
 99.00     99000     500    D    2        Bacteria
 60.00     60000     200    P    1224       Proteobacteria
 45.00     45000     100    C    1236          Gammaproteobacteria
...
```

How do we merge multiple reports to make a table we can then analyse?


## Installing kraut

[Kraut](https://github.com/quadram-institute-bioscience/kraut) (KRAken UTilities) is a small utility developed by QIB Core Bioinformatics and helps combining and plotting the reports. It's a suite of utilities linked to Kraken.

## Merging reports

`kraut make-table` merges multiple Kraken2 reports into a single abundance table, ready for downstream analysis in R or Python. You point it at your report files and it lines them up into a taxa × samples matrix.

```bash
kraut make-table [OPTIONS] INPUT_FILES...
```

The only required argument is one or more Kraken report files. By default it writes to stdout, so you'll usually want `-o`:

```bash
kraut make-table data/*.krep -o species_table.tsv
```

This gives you a species-level table (`-r S` is the default) using cumulative read counts (the `TOT` metric), which is the standard starting point for comparing samples.

#### Choosing what goes in the table

Three options control the content:

**Rank** (`-r`/`--rank`) — pick a taxonomic level: `K`, `P`, `C`, `O`, `F`, `G`, `S`, or `ALL`. Defaults to `S` (species).

**Metric** (`-m`/`--metric`) — how abundance is counted:
- `TOT` — cumulative counts (clade reads, this taxon + descendants) — the default
- `LVL` — taxon-specific counts (reads assigned directly to that taxon)
- `PERCENTAGE` — relative abundance as a percentage

**Identifier** — by default rows are named by taxon. Add `-p`/`--rank-prefix` for Greengenes-style prefixes (`s__Escherichia_coli`), or `--taxid` to use NCBI TaxIDs instead of names (handy when names collide or you're joining on IDs later).

A genus-level relative-abundance table with rank prefixes:

```bash
kraut make-table data/*.krep -r G -p -m PERCENTAGE -o genus_table.tsv
```

#### Cleaning up the table

Two flags trim noise:

```bash
# Drop the unclassified row entirely
kraut make-table data/*.krep --no-unclassified -o table.tsv

# Keep only taxa reaching ≥0.5% in at least one sample
kraut make-table data/*.krep --min-perc 0.5 -o filtered_table.tsv
```

Note that `--min-perc` keeps a row if *any* sample crosses the threshold, so a taxon abundant in one sample won't be dropped just because it's rare in the others.

## Plotting a single sample with `kraut plot-single`

`kraut plot-single` visualises the taxonomic composition of one Kraken report. 
Pass the report and an output path where the extension picks the format — use `.png`, `.svg`, or `.pdf` for a static plot, or `.html` for an interactive one:

```bash
kraut plot-single sample.krep -o composition.png
kraut plot-single sample.krep -r P -o interactive.html   # Phylum-level, interactive
```

Tidy up the plot by folding rare taxa into "Others" with `--min-perc` (default 1.0%) or by keeping only the `--top-taxa N`, and add a `--title` for labelling:

```bash
kraut plot-single sample.krep --min-perc 0.5 --title "Sample A1 Composition" -o plot.svg
```

## Comparing samples with `kraut plot-multi`

`kraut plot-multi` plots taxonomic composition across several Kraken reports at once, as either a stacked bar chart (the default) or a bubble chart via `--kind bubble`. 
As with `plot-single`, the output extension sets the format, so `.html` gives you an interactive plot:

```bash
kraut plot-multi reports/*.krep -o stacked_bars.png
kraut plot-multi reports/*.krep --kind bubble -o bubbles.html
```

The same tidying options apply — `--min-perc`, `--top-taxa`, `--no-unclassified`, and `--title` — so you can, for example, focus on the ten most abundant genera across the cohort:

```bash
kraut plot-multi reports/*.krep -r G --top-taxa 10 -o top_genera.png
```
