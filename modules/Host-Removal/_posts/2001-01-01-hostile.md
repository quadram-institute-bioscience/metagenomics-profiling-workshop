## Using Deacon



Deacon is a host-read removal pipeline that uses its own aligner and index format to filter out contaminant reads.

:bulb: Deacon requires building a custom index from your host reference genome before filtering.

### Downloading coffee

You can use NCBI's `datasets` tool to download the `GCA_036785865.1` reference 
[as described here]({{ site.baseurl }}/{% link modules/Extras/_posts/2001-01-01-ncbi-datasets.md %}).

### Index reference (Deacon)

Note that the FASTA file can be masked to remove regions similar to bacterial segments, but we will skip this step
for the workshop.

Deacon uses its own indexing system. You will need to build an index with this syntax:

```bash
deacon index build --threads 8 --output coffee-index GCA_036785865.1.fna
```

* `GCA_036785865.1.fna` is the path to the FASTA file
* `coffee-index` is the basename for the output index files. You will use this when cleaning the reads

### Clean reads

The *deacon filter* workflow requires the FASTQ files (one if single-end, two if paired-end),
the index you built, and output paths for the cleaned files. When using `--deplete`, Deacon removes reads that align to the host reference.

The `--summary` flag produces a JSON file with statistics about host reads removed.

```bash
deacon filter --deplete --threads 8 \
  --output clean_1.fastq.gz \
  --output2 clean_2.fastq.gz \
  --summary stats.json \
  coffee-index \
  sample_1.fastq.gz \
  sample_2.fastq.gz
```

Replace the filenames with the paths to your actual reads and index.
