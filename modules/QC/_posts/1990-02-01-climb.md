CLIMB is a cloud-based platform that provides access to a Linux environment with pre-installed bioinformatics tools. It is designed to support microbiologists and bioinformaticians in their research, offering a range of resources for data analysis and visualization.

It's particularly convenient for being **browser based**, and for offering in the same environment

* a terminal, with Conda and Nextflow pre-installed, and
* a Jupyter Notebook interface, with R and Python kernels pre-installed.
* RStudio

For the details about CLIMB check the [CLIMB documentation website](https://docs.climb.ac.uk/).

## A first tour of the command line

Each UNIX user has a username, in the case of CLIMB everyone has the same. You can get your username with the command

```bash
whoami
```

A very important command is *print working directory*, that will tell you where in the filesystem you are (in which folder you are operating):

```bash
pwd
```

The output can be something like `/shared/team/users/`

To **list** the files in your current directory:

```bash
ls
```

You can have a detailed view with an extra option `-l` (lowercase L, stands for *long format*)


```bash
ls -l
```


### Your personal working directory

Each participant can personal directory under `/shared/team/users/`, named after your name. 


```bash
mkdir /shared/team/users/andrea_telatin/
```

Move into it using **change directory**:

```bash
cd /shared/team/users/andrea_telatin
```

Check you've arrived with `pwd`.

### Creating and viewing a FASTA file

Create a simple FASTA file with, you can use the left side bar, or the following command

```bash
curl -o demo.fa "https://raw.githubusercontent.com/telatin/learn_bash/refs/heads/master/phage/vir_protein.faa"
```
 

To print the whole file to screen:

```bash
cat demo.fa
```

To see just the beginning or the end of a file:

```bash
head demo.fa
tail demo.fa

head -n 2 demo.fa
```

### Counting lines with a pipe

The pipe symbol `|` sends the output of one command straight into another, without creating an intermediate file:

```bash
cat demo.fa | wc -l
```

Here, `cat` prints the file and `wc -l` ("word count", `-l` = lines) counts how many lines pass through the pipe — so this tells you the total number of lines in `demo.fa`.

### Extracting and counting sequences

In a FASTA file, every sequence header starts with `>`. Use `grep` to pull out lines matching a pattern:

```bash
grep ">" demo.fa
```

Pipe that into `wc -l` to count how many sequences are in the file:

```bash
grep ">" demo.fa | wc -l
```

### Using a Conda environment

Bioinformatics tools are usually kept in dedicated Conda environments to avoid version conflicts. 
Activate the `qc` environment:

```bash
conda activate /shared/team/conda/telatin.cli-giba-2026/qc
```

Now inspect your file with `seqfu stats`:

```bash
seqfu stats demo.fa
```

This reports basic sequence statistics — number of sequences, total length, and min/max/average sequence length — all in a single command.
