[FastQC]({{ site.baseurl }}/{% link modules/QC/_posts/2000-01-20-fastqc.md %}) - that generates a quality report on your files -
and [fastp]({{ site.baseurl }}/{% link modules/QC/_posts/2000-02-01-fastp.md %}) - that will both trim/filter and generate reports are typical tools in the QC realm

[MultiQC](https://seqera.io/multiqc/) is a bit different: it's an **aggregator of results** from different steps.


### Installation

MultiQC is available from Bioconda but also installable as a standard python package via *pip*:
```bash
pip install multiqc
# or
conda install -c bioconda multiqc
```
 
### Basic usage

Run it in a directory containing your tool outputs (logs, JSON, HTML reports — MultiQC scans and detects them automatically):

```bash
multiqc .
```

This produces:
- `multiqc_report.html` — the interactive summary
- `multiqc_data/` — parsed data (TSV files, JSON) behind the report

### Typical workflow example

```bash
# Run FastQC on all your fastq files
fastqc  -o fastqc_out/ DIRECTORY_TO_SCAN...
```

MultiQC will find the FastQC zips and the fastp JSON reports and combine them into one report, with per-sample comparison plots (quality scores, adapter content, duplication, etc.).

Some other useful flags you can use (and combine):

```bash
multiqc . -n my_project_report   # custom output name
multiqc . -f                      # force overwrite existing report
multiqc . -i "GIBA Workshop"      # custom report title
multiqc . --ignore "*/tmp/*"      # exclude paths
```

### Example

You can heavily configure your report with a configuration file like:

```yaml
title: "GIBA metagenomics workshop"
subtitle: "Quadram Institute - Core Bioinformatics"
intro_text: >
  Read QC, host decontamination and taxonomic profiling summary for the coffee fermentation project

report_section_order:
  fastp:
    order: 1000
  deacon:
    order: 900
  kraken:
    order: 800
  metaphlan:
    order: 700

# ---------------------------------------------------------------------------
# 2) MANUALLY ADD VERSIONS FOR KRAKEN2 AND METAPHLAN
# ---------------------------------------------------------------------------
software_versions:
  Kraken:
    Kraken2: "2.17.1"
  MetaPhlAn:
    MetaPhlAn: "4.2.4"
    # Optional: record the reference database version too, for reproducibility.
    # It's not a "software version" but showing it here is a convenient way to
    # keep it attached to the report without extra tooling.
    MetaPhlAn_DB: "mpa_vJan25_CHOCOPhlAnSGB_202503"

```

Then run it as

```bash
multiqc -c config.yaml -o outdir PATH_TO_SCAN
```
