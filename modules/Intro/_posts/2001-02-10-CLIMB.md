## Locations

![CLIMB directories]({{ site.baseurl }}{% link img/tree.png %})

#### Home directory

As in most Unix system, every user has a home directory. The Bash shortcut to it is `~`, or `$HOME`.

Unlike most Unix systems, though, all CLIMB users have the same **username** (jovyan), and thus the
same home directory `/home/jovyan`. 

#### Team share

A special directory we all can write and read from is `/shared/team`, or it's alias in your home `~/shared-team/`.
This is shared across members of a team, in our case the training team.


#### Public share

this is a read-only directory readable from all CLIMB users. For the scope of this training, it contains some Kraken2
databases


## Environments

We will see that the environments created by a user can be activated by all the members of the team, as long as the 
full path is used.

For example:

```
conda activate /shared/team/conda/telatin.cli-giba-2026/qc
```
Will activate an environment with:

- FastQC
- SeqFu
- fastp
- MultiQC
- Deacon
